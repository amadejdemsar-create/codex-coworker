Claude Code plugin

A second opinion,
from a second model.

codex-coworker puts OpenAI Codex (gpt-5.5) inside Claude Code as a coworker, not a black box. Claude drafts a plan, diff, or piece of writing; Codex red-teams it; Claude reconciles the critique in the open and tells you what changed.

The idea

Two models with different blind spots beat one

Claude and Codex share most of their training, so their agreement is weak evidence and their disagreement is the signal worth mining. The plugin is built to surface that disagreement, not paper over it.

Claude
decision owner
  • Does its own thinking first
  • Writes a tight brief for Codex
  • Accepts the good catches, rejects the weak ones with a reason
  • Owns the final call
⇄
Codex
red-team critic
  • Attacks the plan, finds gaps and risks
  • Returns a structured, bounded critique
  • Disagrees when it has a reason
  • Never a co-equal voter

Consensus is not the goal. When a real disagreement cannot be resolved in three rounds, both positions are surfaced to you, so a person decides instead of two models quietly averaging.

Two skills

What you can do with it

Trigger either by name, or just say what you want. Claude picks the right one.

/codex-coworker:review

Get a second opinion

An adversarial review of a plan, a diff, or a piece of writing, run as a bounded loop:

1Claude drafts the work and its own view of it
2Codex returns a strict critique: agreements, disagreements, risks, missing pieces, changes
3Claude reconciles in the open, escalates only a material dispute, hard cap 3 rounds
modes: plan · code · writing · general
say "ask codex", "second opinion", "red-team this", or before a high-stakes call Claude can reach for it on its own
/codex-coworker:image

Generate an image

Render an icon, logo, avatar, or hero through Codex's built-in image tool. The command verifies a real PNG actually landed on disk before it reports success.

codex-image
$ codex-image "a flat-vector navy fox on cream" \
    ./logo.png 1024x1024
# saved ./logo.png (220484 bytes, 1024x1024 PNG)
requires a ChatGPT-account login (codex login), not an API key. codex-doctor tells you which you are on.
Setup

Running in about a minute

You need the Codex CLI installed and logged in. The plugin is two commands on top.

1

Install the Codex CLI pick one

install codex
$ npm install -g @openai/codex      # or
$ brew install --cask codex         # or
$ curl -fsSL https://chatgpt.com/codex/install.sh | sh
2

Log in ChatGPT account, required for image generation

auth
$ codex login
3

Add the marketplace and install the plugin

claude
$ claude plugin marketplace add amadejdemsar-create/codex-coworker
$ claude plugin install codex-coworker@codex-coworker
$ codex-doctor   # confirm binary, auth, model, flags
Make it yours

How to customize it

Sensible defaults out of the box. Every knob is an environment variable, a flag, or a small config snippet.

$CODEX_MODEL

The model Codex reviews with. Defaults to gpt-5.5. Override per call with --model if your account uses a different one.

$CODEX_BIN

Point at a specific codex binary. Useful when the one on your PATH is older than the one you want to use.

--effort

lowquick sanity check
mediumdefault, most reviews
highhard architecture or security

--mode

plandesigns, decisions, roadmaps
codediffs and bug diagnoses (with --cd)
writingcopy, posts, positioning
generalany reasoning to stress-test

Skip the permission prompt

Allowlist the plugin's commands once in ~/.claude/settings.json:

settings.json
{
  "permissions": {
    "allow": [
      "Bash(codex-consult:*)",
      "Bash(codex-doctor:*)",
      "Bash(codex-image:*)"
    ]
  }
}

Make Claude consult Codex automatically

A plugin cannot set a global behavior policy, so to have Claude reach for Codex before high-stakes calls, paste this into your own ~/.claude/CLAUDE.md:

CLAUDE.md
## Codex second opinion
Before committing to a HIGH-STAKES choice (architecture, security, an
irreversible operation, a major refactor, a big strategic decision), use the
/codex-coworker:review skill to have Codex red-team it first, then reconcile
its critique in the open and tell me what changed. Codex is a critic, not a
co-equal voter; I stay the decision owner.
Built to be cheap and safe

Lean by construction

A second model is only worth it if it does not burn your rate limit or your context. Four things keep it tight.

01

Read-only

Critiques run in a read-only sandbox. Codex inspects, it never writes to your files.

02

Bounded output

A strict JSON schema caps the answer, and only that final answer enters Claude's context, never the raw reasoning.

03

One round by default

It escalates to a second or third round only on a genuine, unresolved disagreement. Hard cap three.

04

Rate-aware

A $20 ChatGPT Plus plan allows roughly 15 to 80 gpt-5.5 messages per 5 hours. Defaults are picked to fit inside that.
